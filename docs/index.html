PlanX CartoLab

Native 2.5D building styling, bivariate cartography, cartograms, ridge maps, uncertainty-aware rendering, and print layout polish in one QGIS plugin.

Download release View workflow

Built for maps that need to be read, defended, and published.

CartoLab keeps the cartographic decisions close to the data: height, uncertainty, dual indicators, geometry distortion, raster surfaces, and layout hierarchy.

2.5D Building Style

Native QGIS renderer with height fields, material presets, shadows, wall shading, stepped floors, and QML export.

Bivariate Mapping

NxN colour matrices for two-variable thematic maps with clean post-processing symbology.

Cartograms

Continuous-area distortion for value-proportional polygon outputs.

Ridge Maps

Raster scanlines converted into expressive ridge-line vector maps.

Value-by-Alpha

Reliability and uncertainty encoded as opacity without losing the primary variable.

Adaptive Classes

Geometric Interval, Fisher-Jenks, and Head/Tail Breaks for skewed urban datasets.

Layout Automation

Isometric stacks, bivariate legends, typography hierarchy, and minimalist grids.

Feature Inspector

Floating HTML cards with radar-style attribute charts for quick map reading.

A production workflow inside QGIS.

Use the dashboard for guided work, or launch every algorithm from the Processing Toolbox when building repeatable models.

1

Style the spatial object

Apply 2.5D height-field styling to building polygons and export the QML style for reuse.

2

Classify the indicators

Use adaptive classification to understand skew, heavy tails, and natural breaks before mapping.

3

Encode relationships

Build bivariate, Value-by-Alpha, and cartogram outputs for multidimensional urban interpretation.

4

Finish the layout

Add legends, grids, typography, and isometric stacks for presentation-ready map sheets.

Compact architecture, strong surfaces.

CartoLab stays dependency-light and QGIS-native while exposing polished user workflows.

Core engines

Pure logic for classification, colour matrices, cartograms, affine transforms, and native 2.5D renderer configuration.

Processing provider

Algorithm entry points for repeatable QGIS models and toolbox-driven workflows.

Dashboard UI

A production console for modules, diagnostics, 2.5D styling, layout actions, and feature inspection.
